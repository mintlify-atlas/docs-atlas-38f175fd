## Overview

The Shortcut tool executes keyboard shortcuts using key combinations separated by `+`. Supports modifier keys (Ctrl, Alt, Shift, Win) and special keys for quick actions and system commands.

## Parameters

<ParamField path="shortcut" type="string" required>
  Keyboard shortcut string with keys separated by `+`. Case-insensitive.
  
  Examples: `"ctrl+c"`, `"alt+tab"`, `"win+r"`, `"ctrl+shift+esc"`
</ParamField>

## Return Value

Returns a success message confirming the shortcut was pressed.

Example: `"Pressed ctrl+c."`

## Supported Keys

### Modifier Keys
- `ctrl` - Control key
- `alt` - Alt key
- `shift` - Shift key
- `win` - Windows key

### Special Keys
- `enter`, `return` - Enter key
- `tab` - Tab key
- `esc`, `escape` - Escape key
- `space` - Space bar
- `backspace` - Backspace key
- `delete` - Delete key
- Arrow keys: `up`, `down`, `left`, `right`
- Function keys: `f1` through `f12`
- `home`, `end`, `pageup`, `pagedown`

### Regular Keys
- Letters: `a-z`
- Numbers: `0-9`
- Punctuation: Most standard keyboard symbols

## Examples

### Copy Text

```json
{
  "shortcut": "ctrl+c"
}
```

Copies selected text to clipboard.

### Paste Text

```json
{
  "shortcut": "ctrl+v"
}
```

Pastes clipboard content at cursor position.

### Switch Applications

```json
{
  "shortcut": "alt+tab"
}
```

Cycles to the next open application window.

### Open Run Dialog

```json
{
  "shortcut": "win+r"
}
```

Opens the Windows Run dialog for launching applications.

### Open Task Manager

```json
{
  "shortcut": "ctrl+shift+esc"
}
```

Directly opens Windows Task Manager.

### Select All

```json
{
  "shortcut": "ctrl+a"
}
```

Selects all content in the active window.

### Undo

```json
{
  "shortcut": "ctrl+z"
}
```

Undoes the last action.

### Save File

```json
{
  "shortcut": "ctrl+s"
}
```

Saves the current document or file.

### Open Start Menu

```json
{
  "shortcut": "win"
}
```

Opens the Windows Start menu (single key, no combination).

## Error Handling

<AccordionGroup>
  <Accordion title="Shortcut not recognized">

  </Accordion>
  <Accordion title="Shortcut has no effect">
    The active application may not support this shortcut, or wrong window is focused.
  </Accordion>

## Technical Details

- **Key press timing**: ~50ms delay between key down and key up events
- **Modifier handling**: Modifiers are pressed in order, then released in reverse
- **Implementation**: Uses Windows SendInput API for keyboard simulation
- **Special handling**: Alt+Tab requires special timing for reliable window switching

<Warning>
Some shortcuts like Ctrl+Alt+Delete may be intercepted by the system and cannot be simulated.
</Warning>
