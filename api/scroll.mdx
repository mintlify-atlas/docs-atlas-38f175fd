## Overview

The Scroll tool scrolls vertically or horizontally at specified coordinates, on UI elements by label ID, or at the current mouse position. Useful for navigating long content, lists, and web pages.

## Parameters

<ParamField path="loc" type="list[int]" optional>
  Screen coordinates `[x, y]` where to scroll. If omitted, scrolls at current mouse position.
</ParamField>

<ParamField path="label" type="int" optional>
  Label ID of a scrollable element from Snapshot. Takes precedence over `loc`.
</ParamField>

<ParamField path="type" type="string" default="vertical">
  Scroll direction type:
  - `"vertical"` - Scroll up or down
  - `"horizontal"` - Scroll left or right
</ParamField>

<ParamField path="direction" type="string" default="down">
  Scroll direction:
  - For vertical: `"up"` or `"down"`
  - For horizontal: `"left"` or `"right"`
</ParamField>

<ParamField path="wheel_times" type="int" default="1">
  Number of scroll wheel notches. Each notch scrolls approximately 3-5 lines.
</ParamField>

## Return Value

Returns a success message describing the scroll action performed.

Example: `"Scrolled vertical down by 3 wheel times at (800,400)."`

## Examples

### Scroll Down a Page

```json
{
  "loc": [800, 400],
  "type": "vertical",
  "direction": "down",
  "wheel_times": 5
}
```

Scrolls down 5 wheel notches (approximately 15-25 lines) at coordinates (800, 400).

### Scroll Scrollable Element

```json
{
  "label": 38,
  "direction": "down",
  "wheel_times": 3
}
```

Scrolls the scrollable element with label ID 38 down by 3 notches.

### Scroll Up

```json
{
  "loc": [500, 300],
  "direction": "up",
  "wheel_times": 2
}
```

Scrolls up 2 wheel notches to view earlier content.

### Horizontal Scroll

```json
{
  "loc": [600, 450],
  "type": "horizontal",
  "direction": "right",
  "wheel_times": 4
}
```

Scrolls horizontally to the right in a wide content area.

### Scroll at Current Mouse Position

```json
{
  "direction": "down",
  "wheel_times": 1
}
```

Scrolls down at the current mouse position (no `loc` specified).

## Best Practices

<Steps>
  <Step title="Use scrollable element labels">
    Reference scrollable regions by their label ID from Snapshot for precise scrolling
  </Step>
  <Step title="Add Wait after scrolling">
    Use Wait tool after scroll to allow content to load (especially for infinite scroll)
  </Step>
  <Step title="Verify scroll position">
    Call Snapshot after scrolling to confirm the expected content is visible
  </Step>
  <Step title="Adjust wheel_times as needed">
    Start with small values (1-3) and increase if more scrolling is needed
  </Step>
</Steps>

<Tip>
For web pages with DOM mode, check the scroll percentage in Snapshot output to determine if you've reached the top or bottom.
</Tip>

## Error Handling

<AccordionGroup>
  <Accordion title="Location must be a list of exactly 2 integers [x, y]">
    If providing `loc`, it must be in `[x, y]` format with two numbers.
  </Accordion>
  <Accordion title="Desktop state is empty. Please call Snapshot first">
    When using `label`, call Snapshot first to get scrollable element IDs.
  </Accordion>
  <Accordion title="Failed to find element with label X">
    The label ID is not found. Refresh Snapshot to get current element list.
  </Accordion>
</AccordionGroup>

## Technical Details

- **Scroll amount**: Each `wheel_times` is one notch, typically 3-5 lines or ~120 pixels
- **Timing**: ~50ms delay between scroll events for smooth scrolling
- **Focus**: Automatically moves mouse to target location before scrolling
- **Implementation**: Uses Windows mouse_event API with MOUSEEVENTF_WHEEL

<Info>
Different applications interpret scroll wheel events differently. Some scroll by lines, others by percentage of viewport.
</Info>
