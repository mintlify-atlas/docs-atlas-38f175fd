## Overview

The UIAutomation Wrapper (`uia/`) provides a low-level Python abstraction over the Windows UIAutomation COM API via the `comtypes` library. It handles COM object lifetime, pattern wrapping, event subscriptions, and type conversions.

## Core Components

### uia/core.py

Provides the main `UIAutomation` class:

```python
class UIAutomation:
    """Wrapper around IUIAutomation COM interface."""
    
    def __init__(self):
        # Create COM automation object
        self._automation = CreateObject(
            "{ff48dba4-60ef-4201-aa87-54103eef594e}",  # CLSID_CUIAutomation
            interface=IUIAutomation
        )
    
    def get_root_element(self) -> Element:
        """Get the root element (desktop)."""
        return Element(self._automation.GetRootElement())
    
    def get_focused_element(self) -> Element:
        """Get currently focused UI element."""
        return Element(self._automation.GetFocusedElement())
    
    def element_from_handle(self, hwnd: int) -> Element:
        """Get element from window handle."""
        return Element(self._automation.ElementFromHandle(hwnd))
```

### Element Class

Wraps `IUIAutomationElement` COM interface:

```python
class Element:
    """Wrapper around IUIAutomationElement."""
    
    @property
    def name(self) -> str:
        """Element name (e.g., button text)."""
        return self._element.CurrentName
    
    @property
    def control_type(self) -> int:
        """UIA control type ID."""
        return self._element.CurrentControlType
    
    @property
    def bounding_rectangle(self) -> tuple[int, int, int, int]:
        """Element bounds in screen coordinates."""
        rect = self._element.CurrentBoundingRectangle
        return (rect.left, rect.top, rect.right, rect.bottom)
    
    def find_all(self, condition) -> list[Element]:
        """Find all descendant elements matching condition."""
        array = self._element.FindAll(TreeScope_Descendants, condition)
        return [Element(array.GetElement(i)) for i in range(array.Length)]
```

## Control Types

### uia/controls.py

Provides control-specific helper methods:

```python
def is_button(element: Element) -> bool:
    """Check if element is a button."""
    return element.control_type == UIA_ButtonControlTypeId

def is_edit(element: Element) -> bool:
    """Check if element is a text input."""
    return element.control_type == UIA_EditControlTypeId

def get_control_type_name(control_type_id: int) -> str:
    """Convert control type ID to name."""
    mapping = {
        UIA_ButtonControlTypeId: "Button",
        UIA_EditControlTypeId: "Edit",
        UIA_TextControlTypeId: "Text",
        # ... more mappings
    }
    return mapping.get(control_type_id, "Unknown")
```

## Event Handling

### uia/events.py

Manages UIAutomation event subscriptions:

```python
class EventHandler(IUIAutomationFocusChangedEventHandler):
    """Handler for focus change events."""
    
    def __init__(self, callback):
        self._callback = callback
    
    def HandleFocusChangedEvent(self, sender):
        """Called when focus changes."""
        element = Element(sender)
        self._callback(element)

def subscribe_focus_changed(automation: UIAutomation, callback):
    """Subscribe to focus change events."""
    handler = EventHandler(callback)
    automation._automation.AddFocusChangedEventHandler(None, handler)
    return handler
```

<Info>
The WatchDog service uses event subscriptions to monitor focus changes and invalidate cached UI trees.
</Info>

## COM Object Lifetime

The wrapper handles COM object cleanup:

```python
class Element:
    def __init__(self, com_element):
        self._element = com_element
        # COM object ref count increased
    
    def __del__(self):
        # Release COM reference on garbage collection
        self._element = None
```

<Warning>
Improper COM object lifetime management can cause memory leaks or access violations. The wrapper ensures proper cleanup.
</Warning>

## Error Handling

COM operations can fail. The wrapper converts COM errors to Python exceptions:

```python
from comtypes import COMError

def safe_get_property(element, property_name):
    try:
        return getattr(element._element, property_name)
    except COMError as e:
        if e.hresult == -2147220991:  # Element not available
            return None
        raise
```

## Coordinate Systems

UIAutomation uses screen coordinates:

```python
# All coordinates are in virtual desktop space
# Multi-monitor setups use a single coordinate system
# Example: (2000, 100) may be on the second monitor

element.bounding_rectangle  # Returns (left, top, right, bottom) in screen pixels
```

<Tip>
UIAutomation coordinates match screen coordinates exactly - no DPI scaling needed. This is why Windows-MCP input is reliable across different display settings.
</Tip>

## Performance Considerations

| Operation | Time | Notes |
|-----------|------|-------|
| Get root element | ~1ms | Cached by Windows |
| Find all children | ~10-50ms | Depends on element count |
| Get element property | ~0.1ms | Direct COM call |
| Subscribe to event | ~5ms | One-time setup cost |

## Best Practices

<Steps>
  <Step title="Cache element references">
    Don't repeatedly query the same element - cache the Element object
  </Step>
  <Step title="Use patterns when available">
    Prefer Invoke pattern over simulating clicks for better reliability
  </Step>
  <Step title="Handle COM errors">
    UI elements can disappear - wrap property access in try/except
  </Step>
  <Step title="Release references promptly">
    Set element variables to None when done to release COM objects
  </Step>
</Steps>
